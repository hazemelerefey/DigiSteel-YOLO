2 The Foundation — Optimized Baseline (4A)

The Problem

After Week 3's ablation, we discovered that the old baseline (75.8%) was trained with a suboptimal recipe. The model was undertrained — not because of architecture, but because of training configuration. We rebuilt the baseline from scratch with a systematically optimized recipe.

Key Insight

Training recipe optimization yielded +3.0% mAP — more than any architectural change achieved in Weeks 1-3. The model capacity was always there; we just weren't using it properly.

3 DAFE — Defect-Aware Feature Enhancement

Motivation

Steel surface defects fall into two fundamentally different categories that require different detection strategies:

Linear Defects
Crazing, Scratches
Thin edges, hairline cracks
Require edge detection
vs
Surface Anomalies
Pitting, Scale, Inclusions
Texture irregularities
Require texture analysis

Standard YOLO backbone uses generic convolution filters that are optimized during training but start from random initialization. For steel defects, we can do better — we know a priori that edge detection and texture analysis are the two fundamental operations needed.

Edge Branch — Sobel-Initialized Convolution

The edge branch uses a standard 3×3 convolution, but instead of random initialization, the first two filters are initialized as Sobel-X and Sobel-Y operators — the classical edge detection kernels from computer vision. All weights remain learnable, so the network can adapt these edge detectors during training to the specific characteristics of steel defects.

Why Sobel Initialization
Starts as a known-good edge detector
Crazing cracks are thin edges — perfect match
Scratches are also linear — benefits from edge awareness
Learnable — adapts to steel-specific patterns during training
vs Random Init
No wasted epochs learning basic edge detection
Converges faster on edge-sensitive defects
Provides inductive bias without restricting capacity
First two filters start meaningful, rest use Kaiming

Texture Branch — Local Variance

The texture branch computes local variance (E[X²] − E[X]²) using average pooling, then processes through a 1×1 convolution. Local variance captures texture irregularities — pitting creates rough surfaces, scale creates bumpy textures, inclusions create localized anomalies. These patterns show up as high local variance, distinct from the uniform background.

Channel Attention & Residual

After concatenating both branches, a SE-style channel attention mechanism (squeeze-excitation) re-weights the features, letting the model emphasize the most informative channels. A learnable residual connection with parameter α (initialized to sigmoid(−2.2) ≈ 0.1) ensures the enhancement starts small and grows as the model learns — preventing disruption of pretrained features.

DAFE Placement

DAFE is inserted at P2 (160×160) and P3 (80×80) in the backbone — the two highest-resolution feature maps. This is deliberate:

  • P2 captures fine-grained spatial details — critical for thin crazing cracks
  • P3 provides slightly more semantic features — good for larger defects
  • P4/P5 are too low-resolution for fine defect features — DAFE would add parameters without benefit

5 Crazing — The Hardest Problem

Why Crazing Is Hard

Crazing defects are fine hairline cracks on the steel surface. They are the most challenging class in NEU-DET for several reasons:

Challenge Details Impact
Low contrast Cracks are subtle — only slightly darker than background Hard to distinguish from noise
Thin geometry Cracks are 1-3 pixels wide at 200px resolution Most of the bounding box is background
Large boxes, small defects Mean bounding box area = 22.9% of image IoU penalizes localization errors heavily
Directional variety Cracks can run in any direction Generic filters struggle — need edge-aware features
200px resolution Native image is only 200×200 pixels Upscaling can't recover detail that isn't there

Dataset Analysis

Crazing bounding boxes are wide and thin — mean width 62.6% of image, mean height 37.2%. The boxes span large areas but the actual crack is a thin line running through them. This means the model must learn to detect a thin edge within a large region, while the IoU metric penalizes any misalignment along the full width of the crack.

How DAFE Helps Crazing

DAFE Component Mechanism Effect on Crazing
Sobel-X/Y Init First filters start as horizontal/vertical edge detectors Directly captures crack orientations from epoch 1
Learnable Weights Sobel filters adapt during training Learns crack-specific patterns (width, contrast, direction)
Channel Attention SE mechanism re-weights edge features Emphasizes crack-relevant filter outputs
Residual Connection x + α·enhanced, α starts at 0.1 Gradually introduces edge features without disrupting pretrained backbone

Key Lessons

Attention stacking doesn't work
GhostConv + WFCA + EMA (6 modules) = +0.1%. One well-designed module (DAFE) = +1.5%. More modules ≠ better results.
Larger models overfit on small data
YOLOv11s (9.4M params) on 1290 images = −2.4%. The nano model (2.6M) is the right capacity for this dataset.
Training recipe > architecture
Recipe optimization gave +3.0%. DAFE on top gave +1.5%. Always optimize the baseline first — the model was undertrained, not under-designed.
Domain knowledge beats generic modules
DAFE's Sobel initialization works because we know steel defects are edges and textures. Generic attention modules (WFCA, EMA) don't encode this prior knowledge.
Precision-recall trade-off is acceptable
DAFE is more conservative (85.2% precision, 70.0% recall). For industrial inspection, high precision = fewer false alarms = less wasted human review time.

Decision: TTA Not Counted as Model Performance

TTA is an evaluation technique, not a model improvement. It does not change the learned weights — it only changes how inference is performed. For a fair comparison of model architectures, we evaluate all models with standard inference only. TTA can be applied to any model, so it is not a differentiator. We do not report TTA results as model performance in this report.

Precision-Recall Trade-off

DAFE introduces a notable precision-recall trade-off:

What DAFE Does Well
Precision: 71.9% → 85.2% (+13.3%)
Fewer false positives — more confident detections
Edge features help reject non-defect patterns
For industrial inspection: fewer wasted reviews
The Trade-off
Recall: 76.3% → 70.0% (−6.3%)
More missed defects — some true positives lost
DAFE is conservative — enhances only strong edge/texture signals
For safety-critical: missing defects can be costly

This trade-off is acceptable for our use case. In industrial steel inspection, false positives (flagging good steel as defective) waste human review time and slow production. DAFE's high precision means the model is more trustworthy when it does flag a defect. The recall drop can be addressed in future work through class-weighted loss or multi-dataset training.

Why mAP@0.5:0.95 Dropped

The mAP@0.5:0.95 metric (44.2% vs 45.2% baseline) measures localization quality at higher IoU thresholds. The slight drop suggests that DAFE's edge-focused features help with classification (is this a defect?) more than localization (where exactly is the defect?). This is expected — edge features tell the model what to look for, but precise bounding box regression requires different capabilities. The Inner-WIoU loss already addresses this, and further tuning may help.

8 Conclusions & Week 5 Plan

Week 4 Summary

We rebuilt the baseline from scratch with an optimized training recipe, discovering that the old baseline was undertrained by 3.0%. On top of this stronger foundation, we designed DAFE (Defect-Aware Feature Enhancement) — a novel dual-branch module that uses Sobel-initialized edge detection and local variance texture analysis, purpose-built for steel surface defects. DAFE achieved 80.3% mAP@0.5, breaking the 80% barrier with +1.5% over the optimized baseline and +4.5% over the old baseline.


The most significant result is on crazing detection — the hardest class. From 13.3% (old baseline) to 48.6% (DAFE v4), a +35.3 percentage point improvement. DAFE's Sobel-initialized edge branch directly addresses the fundamental challenge of detecting fine hairline cracks in low-resolution steel images.
