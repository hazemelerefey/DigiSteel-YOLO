- **Verdict:** First architecture that delivers real improvement

## 🔬 DAFE — The Novel Contribution

**DAFE (Defect-Aware Feature Enhancement)** is the core novelty of DigiSteel-YOLO.

### Design Philosophy
Steel surface defects fall into two categories:
1. **Linear defects** (scratches, crazing) → thin edges and cracks
2. **Surface anomalies** (pitting, scale, inclusions) → texture irregularities

DAFE uses a dual-branch design to handle both:

```
Input Features (C channels)
    ├── Edge Branch (Sobel-initialized conv → C/2 channels)
    │   └── Detects linear defects via learned edge filters
    ├── Texture Branch (local variance → C/2 channels)  
    │   └── Detects surface anomalies via local texture statistics
    └── Fusion → Channel Attention → Residual Output
        └── Learnable alpha (sigmoid(-2.2) ≈ 0.1) controls enhancement strength
```

### Key Components
- **EdgeAwareConv:** Conv2d initialized with Sobel-X and Sobel-Y filters. All weights learnable — starts as edge detector, adapts during training.
- **TextureBranch:** Computes local variance (E[X²] − E[X]²) to capture texture irregularities. Lightweight — just AvgPool + 1×1 conv.
- **Channel Attention:** SE-style squeeze-excitation after fusion.
- **Learnable Residual:** `output = x + sigmoid(alpha_raw) * enhanced` — starts small, learns to contribute.

### Why It Works
- Crazing (hardest class): 40.3% → 48.6% (+8.3%) — Sobel initialization directly targets these fine cracks
- Scratches: 97.3% → 98.6% (+1.3%) — edge detection helps
- Inclusion: 82.9% → 86.5% (+3.6%) — texture branch catches irregularities
- Precision: 71.9% → 85.2% (+13.3%) — fewer false positives

### The Trade-off
- Recall: 76.3% → 70.0% (−6.3%) — DAFE is more conservative, misses some true positives
- mAP@0.5:0.95: 45.2% → 44.2% (−1.0%) — slightly worse at higher IoU thresholds

### Failed Experiments (What NOT to Do)

| Change | Impact | Why |
|--------|--------|-----|
| Optimized training recipe (AdamW, lr0=0.001, 600ep, copy_paste) | **+3.0%** | Recipe optimization was the biggest single improvement |
| DAFE at P2+P3 | **+1.5%** | Defect-aware feature enhancement on top of strong baseline |
| imgsz=800 | Significant | 4× upscale from 200px preserves fine features |
| mosaic=0 | Required | Mosaic at 200→800px blurs fine crazing features |
| copy_paste=0.1 | Helped | Augmentation for small datasets |
| Inner-WIoU loss | Kept | Aspect-ratio-aware bounding box loss |

1. **Fix the recall drop** (70.0% vs 76.3% baseline)
   - DAFE is too conservative — misses true positives
   - Options: adjust alpha initialization, lower channel attention threshold, or add recall-focused loss term

2. **Improve mAP@0.5:0.95** (44.2% vs 45.2% baseline)
   - Higher IoU threshold means localization needs improvement

## 🏆 Key Decisions & Lessons

1. **Training recipe > architecture** — The optimized recipe alone gave +3%. DAFE on top gave +1.5%. Always optimize the baseline first.

2. **Attention stacking doesn't work** — GhostConv + WFCA + EMA (6 modules) = +0.1%. One well-designed module (DAFE) = +1.5%.

3. **TTA is evaluation, not improvement** — TTA gives +2.8% but doesn't change the model. We don't count it.

4. **Training CSV metrics lie** — Fresh `model.val()` is the gold standard. Training CSV can inflate by 1-2%.

5. **200px images need special treatment** — Mosaic ruins fine features. imgsz=800 (4× upscale) is essential.

6. **Larger model ≠ better on small data** — YOLOv11s overfit on 1200 images. Stick with v11n.
