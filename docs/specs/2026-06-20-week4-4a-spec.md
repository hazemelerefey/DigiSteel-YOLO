# Experiment 4A — "Free mAP" Recipe Spec

## Goal
Train a fresh YOLOv11n from COCO pretrained weights with an optimized training recipe.
Target: **78-80% mAP@0.5** on NEU-DET (currently 75.8%).

## Critical Rules
1. **NO custom modules** — no GhostConv, WFCA, EMA, CoordAttention, DAFE
2. **Standard YOLOv11n** — vanilla Ultralytics model, pretrained on COCO
3. **Every cell must run independently** — no hidden state between cells
4. **All paths must use raw strings or forward slashes** — Windows paths
5. **No fake code** — every API call must match Ultralytics 8.3.x API exactly
6. **Verify everything** — check files exist, check GPU, check dataset before training

## Training Recipe

### Hyperparameters
```yaml
# Model
model: yolo11n.pt  # COCO pretrained, auto-downloaded by Ultralytics
task: detect

# Training
epochs: 600
patience: 150
batch: 16
imgsz: 800  # Base size, multi-scale handles the rest
device: 0   # GPU

# Optimizer
optimizer: AdamW
lr0: 0.001
lrf: 0.01    # Final LR = lr0 * lrf
momentum: 0.937
weight_decay: 0.0005
warmup_epochs: 5
warmup_momentum: 0.8
warmup_bias_lr: 0.1

# Augmentation (THE KEY CHANGES)
mosaic: 0.0        # DISABLED — was sabotaging fine features
mixup: 0.15        # Light mixup
copy_paste: 0.1    # Copy-paste augmentation
degrees: 10.0
translate: 0.2
scale: 0.6
shear: 5.0
perspective: 0.0
flipud: 0.5
fliplr: 0.5
hsv_h: 0.015
hsv_s: 0.7
hsv_v: 0.4
erasing: 0.4

# Multi-scale training
# NOTE: Ultralytics doesn't have a built-in multi-scale param.
# We implement it manually by setting imgsz=800 and relying on
# the scale augmentation (scale: 0.6) to simulate multi-scale.
# Alternative: override batch preprocessing (advanced).

# Label smoothing
label_smoothing: 0.01

# Cosine LR
cos_lr: True

# Other
close_mosaic: 0     # Since mosaic=0, this is irrelevant
amp: True           # Mixed precision
seed: 42
workers: 8
save: True
save_period: 50     # Save checkpoint every 50 epochs
plots: True
```

### Important Ultralytics API Notes
- `model.train()` accepts these as keyword arguments or via a dict
- `cos_lr` is a valid parameter in Ultralytics 8.3.x
- `copy_paste` is valid for detection (uses segmentation-style copy-paste)
- `label_smoothing` is valid
- `close_mosaic` controls when mosaic shuts off (0 = never use mosaic, which is what we want)
- DO NOT use `multi_scale=True` — that's a different thing (inference-time multi-scale)
- The `model` param should be `'yolo11n.pt'` — Ultralytics auto-downloads from hub

## Common Mistakes to Avoid
1. Using `from ultralytics import YOLO` then `YOLO('yolo11n.yaml')` — that creates from scratch, NOT pretrained
2. Using `YOLO('yolo11n.pt')` IS correct — loads pretrained weights
3. Don't mix custom YAML with pretrained weights — use the .pt file directly
4. `close_mosaic=0` when `mosaic=0` is fine — it just means "don't close mosaic since we never opened it"
5. Don't use `model.train(data=..., epochs=...)` positional args — use keyword args
6. Don't forget `data=` must be a STRING path, not a Path object
7. `copy_paste` augmentation requires the dataset to have segmentation masks — if it fails, set to 0.0 and note it
8. Don't create a trainer.py or custom trainer — use vanilla `model.train()`
